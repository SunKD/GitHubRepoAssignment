What is Git?

Git is a version control system; you can think of it as a series of snapshots(commits) of your code base. With the creating of eash snapshot you build our a whole application, seeing the growth of a full app. During this time you can make abrances to experiment and come back to a prefious snapshots you too or merge those experiments in to the your app.

What is Github?

Github is a code hositng platform for version control and collaboration. It lets your and others work together on projects from anywhere using those code snapshots that you took during developemnt.

Git init

Git init is the act of registering this current code base.

Git add

Git add is how we stage the intergration of our current code base into our current list of snapshots

Git status

Git status allows us to see the changes we are going to make

Git commit

Git commit; finally we can commit those changes we added to our last snapshot giving us a new shapshot of our code with whaever new changes were made

Git remote add

Git remote add is used to set up a online repository where we can place all of our current code. This is where the outside world can see our code base.

Git push

Git push is now we get our code to Github, an excellent place to store all of our repositories. Other developers might look at our existing projects to see how we approached problems, pull down our code and add new features or even update our code base with features they wanted to add
